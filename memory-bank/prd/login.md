# PRD — 로그인 페이지

> 마지막 갱신: 2026-05-20

## 1. 개요

### 목적

사용자가 계정에 안전하게 접근하고, 로그인 이후 관심 카드와 컬렉션 같은 개인화 기능을 사용할 수 있도록 한다.

### 대상 사용자

- 회원가입을 완료한 사용자.
- 아직 계정이 없어 이메일 인증 기반 회원가입이 필요한 사용자.
- 관심 카드 또는 컬렉션 관리를 원하는 사용자.
- 가격 추적 기능을 사용하려는 사용자.

### 진입 경로

- 홈페이지 우측 상단 로그인 버튼.
- 상품 상세 페이지에서 관심 카드 등록 시도.
- 컬렉션 또는 개인화 기능 접근 시도.

### 다음 이동 경로

- 로그인 성공 시 홈페이지.
- 이전에 접근하려던 페이지가 있는 경우 해당 페이지로 복귀.

## 2. 목표와 기능 정의

### 목표

- 사용자가 안전하게 계정에 접근할 수 있다.
- 로그인 이후 개인화 기능을 사용할 수 있다.
- 인증 실패와 입력 오류를 명확하게 안내한다.

### 핵심 기능

- 이메일/비밀번호 로그인.
- 이메일/비밀번호 회원가입.
- 회원가입 후 인증 메일 확인 안내.
- 필수 입력값 유효성 검사.
- 로그인 실패 처리.
- 로그인 성공 후 리다이렉트.
- 인증 상태에 따른 공개 페이지 헤더 버튼 전환.
- 로그아웃.

## 3. 사용자 스토리

- As a 회원가입을 완료한 사용자로서  
  I want to 내 계정으로 로그인하고 싶다  
  So that 관심 카드와 컬렉션 정보를 관리할 수 있다.

- As a 가격 추적을 원하는 사용자로서  
  I want to 로그인 후 관심 카드를 저장하고 싶다  
  So that 이후에도 카드 가격 변동을 쉽게 확인할 수 있다.

## 4. 기능 요구 사항

### P0: 필수 요구 사항

- 이메일 입력 필드를 제공한다.
- 비밀번호 입력 필드를 제공한다.
- 로그인 버튼을 제공한다.
- 회원가입 페이지 이동 링크를 제공한다.
- 필수 입력값 유효성 검사를 수행한다.
- 로그인 실패 시 에러 메시지를 노출한다.
- 로그인 요청 중 버튼을 비활성화한다.
- 로그인 성공 시 홈페이지 또는 이전 페이지로 이동한다.
- 공개 페이지 헤더는 비로그인 사용자에게 `로그인`과 `가입하기`를 표시한다.
- 공개 페이지 헤더의 로그인/가입 링크는 현재 내부 경로를 `next`로 보존한다. 단, 현재 경로가 `/`이면 `next`를 붙이지 않는다.
- 로그인 사용자는 공개 페이지 헤더에서 `로그아웃` 버튼만 볼 수 있다.
- 로그아웃 버튼은 Supabase Auth 세션 종료를 시도한 뒤 성공/실패와 관계없이 홈페이지(`/`)로 이동한다.
- 회원가입 페이지는 이메일, 비밀번호, 비밀번호 확인 필드를 제공한다.
- 회원가입 성공 시 즉시 로그인시키지 않고 인증 메일 확인 안내를 노출한다.
- 인증 메일 링크 확인 성공 시 홈페이지 또는 이전 페이지로 이동한다.

### P1: 권장 요구 사항

- 비밀번호 보기/숨기기.
- 로그인 상태 유지.
- 소셜 로그인.
- 비밀번호 찾기.
- 회원가입 페이지 이동 링크.

## 5. 상태 정의

- 기본 상태: 이메일/비밀번호 입력 가능.
- 로딩 상태: 로그인 요청 중 버튼 비활성화.
- 에러 상태: 계정 정보 불일치 또는 네트워크 오류.
- 인증 상태: 이미 로그인된 사용자는 홈페이지로 이동.
- 공개 헤더 비로그인 상태: `로그인`, `가입하기` 링크 노출.
- 공개 헤더 로그인 상태: `로그아웃` 버튼만 노출.

## 6. 결정 대기

- 소셜 로그인의 MVP 포함 여부.

## 7. 결정 사항

- 인증 수단은 Supabase Auth를 사용한다.
- 회원가입은 Supabase Auth 이메일 인증 방식으로 처리한다. MVP에서는 이름, 닉네임, 프로필 테이블, 약관 체크박스, 소셜 회원가입을 제외한다.
- 공개 페이지 헤더의 인증 상태 확인은 Supabase Auth `getClaims()`를 기준으로 한다.
- 로그아웃 후 이동 경로는 항상 홈페이지(`/`)로 한다.
