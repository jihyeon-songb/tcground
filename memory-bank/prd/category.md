# PRD — 카테고리 대분류/소분류 페이지

> 마지막 갱신: 2026-05-22

## 1. 개요

### 목적

사용자가 카드명을 정확히 몰라도 TCG 종류, 세트, 카드 타입, 레어도 등의 카테고리를 통해 카드를 탐색할 수 있도록 한다.

### 대상 사용자

- 특정 TCG 카테고리를 탐색하는 사용자.
- 세트/확장팩 단위로 카드 가격을 확인하려는 사용자.
- 원하는 카드명을 정확히 알지 못하지만 범주 기반 탐색을 원하는 사용자.

### 진입 경로

- 홈페이지 주요 카테고리.
- 헤더 카테고리 메뉴.
- 검색 결과 내 카테고리 링크.

### 다음 이동 경로

- 소분류 카테고리 페이지.
- 상품 상세 페이지.
- 검색 결과 페이지.

## 2. 목표와 기능 정의

### 목표

- 사용자가 대분류와 소분류를 통해 원하는 카드군을 탐색할 수 있다.
- 카테고리별 카드 가격 흐름을 비교할 수 있다.
- 현재 선택한 탐색 범위를 명확하게 인지할 수 있다.

### 카테고리 구조 예시

- 대분류: 포켓몬 카드, 유희왕, 매직 더 개더링.
- 소분류: 세트/확장팩, 카드 타입, 레어도.

## 3. 사용자 스토리

- As a 특정 TCG를 수집하는 사용자로서  
  I want to 게임별 카테고리에서 카드를 탐색하고 싶다  
  So that 내가 관심 있는 카드군의 시세를 확인할 수 있다.

- As a 세트 단위로 카드를 모으는 컬렉터로서  
  I want to 확장팩별 카드 목록을 보고 싶다  
  So that 해당 세트의 주요 카드 가격을 비교할 수 있다.

## 4. 기능 요구 사항

### P0: 필수 요구 사항

- 대분류 카테고리 목록을 제공한다.
- 대분류 선택 시 소분류 목록을 제공한다.
- 선택한 카테고리에 해당하는 카드 목록을 노출한다.
- 카드 목록의 기본 이미지는 한국판(`ko/KR`) printing과 한국 포켓몬센터 이미지가 있으면 이를 우선 노출한다.
- 현재 선택된 카테고리를 표시한다.
- 카드별 가격 요약 정보를 노출한다.
- 카드 클릭 시 상품 상세 페이지로 이동한다.
- 상품이 없는 카테고리의 빈 상태를 제공한다.

### P1: 권장 요구 사항

- 카테고리별 카드 개수 표시.
- 카테고리 내 정렬 기능. 기본 추천순은 가격 데이터가 있는 카드를 먼저 노출하고, 가격 데이터가 있는 카드끼리는 가격 표본 수가 많은 순을 우선한다.
- 레어도/세트 다중 선택 필터: 사이드바 체크박스로 즉시 URL 쿼리(`?rarity=SAR,AR&set=pokemon-kr-151,...`)와 동기화하고, 옵션 목록은 게임 전체 기준으로 항상 노출한다. 별도의 "등록 세트" 그리드/세트 전용 페이지는 두지 않는다.
- 가격대 필터.
- breadcrumbs 제공.
- 카테고리별 가격 상승 카드 표시.

## 5. 상태 정의

- 기본 상태: 선택된 카테고리와 카드 목록 노출.
- 로딩 상태: 카테고리/카드 목록 로딩.
- 빈 상태: "이 카테고리에 등록된 카드가 없습니다" 안내.
- 에러 상태: 카테고리 데이터 조회 실패 안내.

## 6. 필요한 데이터

- categoryId: 카테고리 고유 ID.
- categoryName: 카테고리 이름.
- parentCategoryId: 상위 카테고리 ID.
- cardCount: 카테고리 내 카드 수.
- cardId: 카드 고유 ID.
- cardName: 카드명.
- thumbnailUrl: 카드 이미지.
- avgPrice: 평균 거래가.
- minPrice: 최저가.
- maxPrice: 최고가.
