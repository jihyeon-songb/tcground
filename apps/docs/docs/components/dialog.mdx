# Dialog

사용자의 주의를 잡아 단일 결정을 받아내야 할 때 사용하는 모달 컴포넌트입니다.
열릴 때 첫 focusable 요소로 focus 가 이동하고, Tab 은 다이얼로그 안에서 순환하며, ESC 와 Overlay 클릭으로 닫힙니다. 닫히면 trigger 로 focus 가 복원됩니다.

## 설치

```bash
pnpm add @tcground/headless-ui
```

```ts
import '@tcground/headless-ui/theme.css';
```

## 사용법

Dialog 는 compound 컴포넌트 모음입니다. `Dialog.Root` 안에 trigger, overlay, content, title/description, close 를 조합합니다.

```tsx
import { Dialog } from '@tcground/headless-ui';

export function Example() {
  return (
    <Dialog.Root>
      <Dialog.Trigger>열기</Dialog.Trigger>
      <Dialog.Overlay />
      <Dialog.Content>
        <Dialog.Title>제목</Dialog.Title>
        <Dialog.Description>설명</Dialog.Description>
        <Dialog.Close>닫기</Dialog.Close>
      </Dialog.Content>
    </Dialog.Root>
  );
}
```

`Content` 는 `document.body` 로 portal 됩니다. 같은 페이지의 다른 stacking context 영향을 받지 않습니다.

### Dialog.Overlay / Dialog.Title / Dialog.Description / Dialog.Portal

각각 native `<div>` / `<h2>` / `<p>` 의 props 를 그대로 받습니다. Title 과 Description 은 자동 생성된 id 로 Content 의 aria-labelledby / aria-describedby 와 연결됩니다.

## 접근성

- Content 는 `role="dialog"` 와 `aria-modal="true"` 를 가집니다.
- Title 과 Description 은 자동 생성된 id 로 Content 와 연결됩니다.
- 열릴 때 Content 안의 첫 focusable 요소로 focus 가 이동하고, 닫히면 trigger 로 복원됩니다. 직전 active 요소가 trigger 외에 있으면 그 요소로 돌아갑니다.
- Tab 은 Content 안에서 순환합니다 (focus trap).
- ESC 키와 Overlay 클릭은 Dialog 를 닫습니다.
- 열려 있는 동안 `document.body` 의 overflow 가 hidden 으로 잠겨 배경 스크롤이 막힙니다.
