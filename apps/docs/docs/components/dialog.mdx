# Dialog

사용자의 주의를 잡아 단일 결정을 받아내야 할 때 사용하는 모달 컴포넌트입니다.
portal, trigger/content ARIA 연결, focus trap, ESC/overlay 닫기, focus 복원 같은 동작은 unstyled 패키지 `@tcground/headless`가 담당하고, `@tcground/ui`는 그 위에 스타일을 입힌 styled 레이어입니다. `theme.css`만 불러와도 overlay, content, title, description, close button의 기본 스타일이 포함됩니다.

## 설치

```bash
pnpm add @tcground/ui
```

```ts
import '@tcground/ui/theme.css';
```

## 사용법

Dialog 는 compound 컴포넌트 모음입니다. `Dialog` 안에 trigger, content, title/description, close 를 조합합니다.
`DialogContent`는 내부에서 portal, overlay, 기본 close button을 함께 렌더합니다.

```tsx
import {
  Button,
  Dialog,
  DialogClose,
  DialogContent,
  DialogDescription,
  DialogTitle,
  DialogTrigger,
} from '@tcground/ui';

export function Example() {
  return (
    <Dialog>
      <DialogTrigger asChild>
        <Button>열기</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogTitle>제목</DialogTitle>
        <DialogDescription>설명</DialogDescription>
        <DialogClose asChild>
          <Button>닫기</Button>
        </DialogClose>
      </DialogContent>
    </Dialog>
  );
}
```

`DialogContent` 는 `document.body` 로 portal 됩니다. 같은 페이지의 다른 stacking context 영향을 받지 않습니다.
외부 overlay나 custom close button이 필요하지 않으면 `DialogOverlay`를 직접 렌더하지 않아도 됩니다.

### DialogOverlay / DialogTitle / DialogDescription / DialogPortal / DialogHeader / DialogFooter

각각 대응되는 native element props 를 받습니다. Title 과 Description 은 자동 생성된 id 로 Content 의 aria-labelledby / aria-describedby 와 연결됩니다.

## 접근성

- Content 는 `role="dialog"` 와 `aria-modal="true"` 를 가집니다.
- Title 과 Description 은 자동 생성된 id 로 Content 와 연결됩니다.
- 열릴 때 Content 안의 첫 focusable 요소로 focus 가 이동하고, 닫히면 trigger 로 복원됩니다. 직전 active 요소가 trigger 외에 있으면 그 요소로 돌아갑니다.
- Tab 은 Content 안에서 순환합니다 (focus trap).
- ESC 키와 Overlay 클릭은 Dialog 를 닫습니다.
- 열려 있는 동안 `document.body` 의 overflow 가 hidden 으로 잠겨 배경 스크롤이 막힙니다.
