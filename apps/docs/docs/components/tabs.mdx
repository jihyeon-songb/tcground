# Tabs

여러 개의 관련된 패널 중 하나만 표시하는 컴포넌트입니다.
`@tcground/ui`가 직접 구현한 접근성 primitive로, `theme.css`만 불러와도 list, trigger, active, panel의 기본 스타일이 포함됩니다.
내부에서 tablist/tab/tabpanel ARIA 연결, roving tabindex, 방향키 이동과 활성화를 처리합니다.

## 설치

```bash
pnpm add @tcground/ui
```

```ts
import '@tcground/ui/theme.css';
```

## 사용법

`Tabs` 의 `defaultValue` 와 각 `TabsTrigger` / `TabsContent` 의 `value` 가 짝을 이뤄야 합니다. 매칭되지 않으면 어떤 패널도 표시되지 않습니다.

```tsx
import { Tabs, TabsContent, TabsList, TabsTrigger } from '@tcground/ui';

export function Example() {
  return (
    <Tabs defaultValue='a'>
      <TabsList aria-label='섹션'>
        <TabsTrigger value='a'>A</TabsTrigger>
        <TabsTrigger value='b'>B</TabsTrigger>
      </TabsList>
      <TabsContent value='a'>A 내용</TabsContent>
      <TabsContent value='b'>B 내용</TabsContent>
    </Tabs>
  );
}
```

`TabsList` 에 `aria-label` 을 주어 보조 기기가 tablist 의 목적을 식별할 수 있게 하는 것이 권장됩니다.

### TabsList

native `<div>` 의 props 를 받습니다. 컴포넌트가 추가하는 것:

- `role="tablist"` 자동 적용
- 좌우 방향키 처리: `ArrowRight` / `ArrowLeft` 로 다음 / 이전 trigger 로 focus + activate

`aria-label` 또는 `aria-labelledby` 를 명시해 tablist 의 목적을 알리는 것이 권장됩니다.

## 접근성

- List 는 `role="tablist"` 를 가집니다. `aria-label` 또는 `aria-labelledby` 를 함께 지정하는 것이 권장됩니다.
- Trigger 는 `role="tab"`, `aria-selected`, `aria-controls` 를 가집니다.
- Panel 은 `role="tabpanel"`, `aria-labelledby` 를 가집니다.
- 키보드 지원:
  - `Tab` 으로 tablist 에 진입하면 활성 trigger 에 focus 가 들어갑니다 (roving tabindex).
  - 좌우 방향키 (`ArrowLeft` / `ArrowRight`) 로 trigger 사이를 이동하면 자동으로 해당 탭이 활성화됩니다.
- 비활성 패널의 자식은 unmount 되어 보조 기기 트리에서 제거됩니다.
