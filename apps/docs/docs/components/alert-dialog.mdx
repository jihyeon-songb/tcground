# Alert Dialog

사용자가 실수로 실행하면 안 되는 작업을 한 번 더 확인할 때 사용하는 모달 컴포넌트입니다.
Radix Alert Dialog primitive 기반이며, destructive action, cancel action, title, description을 명확하게 분리합니다.

      <AlertDialogTitle>관심 카드에서 제외할까요?</AlertDialogTitle>

      <AlertDialogAction variant="destructive">삭제</AlertDialogAction>
    </AlertDialogFooter>
  </AlertDialogContent>
</AlertDialog>`}

## 사용법

`AlertDialog`는 `Dialog`보다 확인/취소 흐름에 더 특화되어 있습니다. 삭제, 알림 중지, 대량 변경처럼 사용자의 의도가 확실해야 하는 동작에 사용합니다.

```tsx
import {
  AlertDialog,
  AlertDialogAction,
  AlertDialogCancel,
  AlertDialogContent,
  AlertDialogDescription,
  AlertDialogFooter,
  AlertDialogHeader,
  AlertDialogTitle,
  AlertDialogTrigger,
  Button,
} from '@tcground/ui';

export function Example() {
  return (
    <AlertDialog>
      <AlertDialogTrigger asChild>
        <Button variant='destructive'>삭제</Button>
      </AlertDialogTrigger>
      <AlertDialogContent>
        <AlertDialogHeader>
          <AlertDialogTitle>삭제할까요?</AlertDialogTitle>
          <AlertDialogDescription>이 작업은 되돌릴 수 없습니다.</AlertDialogDescription>
        </AlertDialogHeader>
        <AlertDialogFooter>
          <AlertDialogCancel>취소</AlertDialogCancel>
          <AlertDialogAction variant='destructive'>삭제</AlertDialogAction>
        </AlertDialogFooter>
      </AlertDialogContent>
    </AlertDialog>
  );
}
```

### Controlled

<ComponentPreview
  code={`const [open, setOpen] = useState(false);

<Button variant="outline" onClick={() => setOpen(true)}>

</Button>

  <AlertDialogContent>
    <AlertDialogHeader>
      <AlertDialogTitle>제어된 Alert Dialog</AlertDialogTitle>
      <AlertDialogDescription>
        상위 컴포넌트가 확인 창의 open 상태를 직접 관리합니다.

    </AlertDialogHeader>
    <AlertDialogFooter>
      <AlertDialogCancel>취소</AlertDialogCancel>
      <AlertDialogAction>확인</AlertDialogAction>
    </AlertDialogFooter>
  </AlertDialogContent>
</AlertDialog>`}
>

### AlertDialogTrigger / Header / Footer / Media / Title / Description

대응되는 Radix primitive 또는 native element props를 그대로 받습니다. `AlertDialogTrigger`는 `asChild`를 지원합니다.

## 접근성

- Content는 alert dialog role과 modal semantics를 Radix primitive에서 제공합니다.
- Title과 Description을 함께 배치해 사용자가 결정의 결과를 이해할 수 있게 합니다.
- Cancel action은 항상 명확한 텍스트로 제공하고, destructive action은 시각적으로 구분합니다.
- ESC 키와 cancel action으로 닫을 수 있으며, 닫힌 뒤 focus는 trigger로 복원됩니다.
