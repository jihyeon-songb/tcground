# Dialog — focus trap과 focus 복원

모달 다이얼로그의 핵심 접근성 패턴은 **focus 관리**입니다. 열리면 focus가 다이얼로그 안으로 들어가고,
`Tab` 은 다이얼로그 내부에서만 순환해야 하며(focus trap), 닫히면 원래 트리거로 focus가 되돌아와야 합니다.
`@tcground/headless` 의 `Dialog` 는 여기에 `role="dialog"`, `aria-modal`, `aria-labelledby`, Esc 닫기까지 기본 제공합니다.

## 직접 확인

키보드만으로 두 데모를 비교해 보세요.

- ✅ 버전: `Tab` 으로 트리거에 도달 → `Enter` 로 열기 → `Tab` 이 다이얼로그 안에서만 순환 → `Esc` 로 닫기 → focus 가 트리거로 복귀.
- ❌ 버전: 트리거에 `Tab` 으로 도달조차 안 되고, 열어도 `Tab` 이 배경으로 새며 `Esc` 가 동작하지 않습니다.

## headless가 보장하는 것

- `DialogContent` 는 `role="dialog"`(또는 `alertdialog`), `aria-modal="true"` 를 가집니다.
- `DialogTitle` / `DialogDescription` 이 `aria-labelledby` / `aria-describedby` 로 자동 연결됩니다.
- 열릴 때 직전 focus 요소를 기억하고, 닫힐 때 그 요소로 focus 를 복원합니다.
- `Tab` 은 다이얼로그 내부에서 순환하며, 모달 동안 배경은 `aria-hidden` 처리됩니다.
- `Esc` 로 닫을 수 있습니다.
