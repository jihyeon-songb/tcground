# SegmentedControl — tab 오용과 radiogroup 의미

값을 골라 **같은 뷰를 재구성**하는 세그먼트 UI는 흔히 `role="tablist"` 로 잘못 표시됩니다.
하지만 tab 패턴은 각 tab 이 `aria-controls` 로 **tabpanel** 을 가리켜야 성립합니다.
가리킬 패널이 없는데 tab 이라 선언하면, 스크린리더는 "탭"이라 읽지만 이동할 패널이 없고
방향키도 동작하지 않습니다. 이런 UI의 올바른 의미는 **단일 선택(radiogroup)** 입니다.
`SegmentedControl` 은 `@tcground/headless` 의 `RadioGroup` 위에 세워져 그룹 role 과
roving tabindex, 방향키 선택을 기본 제공합니다.

## 직접 확인

- ✅ 버전: `Tab` 한 번으로 그룹에 진입(선택된 항목에 focus) → 방향키로 선택을 옮김.
- ❌ 버전: 방향키가 동작하지 않고, 항목마다 `Tab` 을 눌러야 하며, "탭"이라 안내되지만 전환되는 패널이 없어 의미가 어긋납니다.

## tab 과 언제 나뉘나

- **SegmentedControl(radiogroup):** 값을 골라 같은 뷰를 다시 구성 — 차트 기간, 정렬 기준.
- **[Tabs](../components/tabs)(tablist):** 각 tab 이 서로 다른 tabpanel 을 전환할 때.

## headless가 보장하는 것

- `SegmentedControl` 은 `role="radiogroup"` 을 가지며 `aria-label` 로 목적을 알립니다.
- 각 항목은 `role="radio"`, `aria-checked` 를 가집니다.
- roving tabindex 로 그룹 전체가 하나의 `Tab` 정지점이 됩니다.
- 방향키(`ArrowUp`/`ArrowDown`/`ArrowLeft`/`ArrowRight`)로 선택을 이동하며 끝에서 순환합니다.
