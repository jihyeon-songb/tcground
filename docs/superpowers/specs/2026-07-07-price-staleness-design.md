# 판매중 호가 부재 시 거래가 표시 — 신선도 명시 설계

날짜: 2026-07-07
상태: 승인됨 (구현 대기)

## 문제

카드 상세는 "평균 거래가"로 eBay Browse 등에서 수집한 **판매중 호가**의
일일 집계(`card_price_snapshots`)를 보여준다. 특정 날짜에 판매중 물량이 없으면
그날 스냅샷 행이 생성되지 않는다. 사용자 인식: "판매중이 없더라도 거래가는
존재해야 한다."

실제 코드 동작을 확인한 결과, 거래가는 이미 사라지지 않는다.
`derivePriceDisplayFromHistory`는 추세 series의 **마지막 점**을 요약가로
사용하므로, 빈 날에는 마지막 실측일의 값이 그대로 이월(carry-forward)된다.

따라서 진짜 문제는 두 가지다:

1. **요약가 신선도** — 이월된 값이 "오늘 값"인지 "며칠 전 값"인지 사용자가
   구분할 수 없다. 오래된 호가가 현재가처럼 보인다.
2. **데이터 정합성 인식** — 차트가 마지막 실측점에서 끝나 버려, 그 값이 계속
   유효한지 아니면 단지 신규 데이터가 없는 건지 불투명하다.

## 접근

**읽기 계층에서 이월을 유지하되, 이월이라는 사실을 투명하게 표시한다.**
DB에는 실측 관측만 남긴다 (조작 행 없음). 검토했으나 채택하지 않은 대안:

- **B (DB 이월 행)**: cron이 빈 날에 전일 값을 복사해 삽입. 조작 데이터가
  실측과 섞이고 행 수가 카드×일수로 폭증하며, 차트·알림에서 재필터링이 필요.
  "매일 행이 존재해야 한다"는 실제 요구가 아니라 "사용자에게 가격이 항상
  보여야 한다"가 실제 요구이며, 그건 이월로 이미 충족됨. 반려.
- **C (`card_printings` 최신가 비정규화 컬럼)**: 유효하지만 별도 목표(정렬용
  DB 페이지네이션, Phase 3)에 속함. 이 스펙 범위 밖.

## 변경 사항

### 1. `PriceDisplay`에 신선도 필드 추가 — `lib/tcg-catalog.ts`

`PriceDisplay` 인터페이스에 필드 하나 추가:

```ts
/** 마지막 실측 스냅샷 이후 경과 일수. 0 = 오늘 데이터. */
stalenessDays: number;
```

`derivePriceDisplayFromHistory`에서 계산:

- `latest.date`(추세 series 마지막 점의 날짜)와 오늘 날짜의 차이를 일 단위로.
- 계산은 순수 함수여야 하므로 "오늘"을 인자로 주입한다. 시그니처를
  `derivePriceDisplayFromHistory(history, today = new Date())` 형태로 확장하고,
  날짜 차이는 UTC 자정 기준 일수로 계산한다 (`toSnapshotDate` 재사용 가능).
- 이 함수는 상세·리스트·알림 세 소비자가 공유하므로 한 곳 수정으로 전부 반영됨.
  알림(`lib/alerts/evaluate.ts`)은 필드가 생겨도 **동작 변화 없음** — 지금은
  읽지 않는다.

### 2. 상세 페이지 요약가 신선도 배지 — `app/cards/[cardId]/page.tsx`

"평균 거래가" 금액 옆(기존 변동률 칩과 같은 줄)에 신선도 배지 추가:

| stalenessDays | 표시 |
|---|---|
| 0 | 배지 없음 (오늘 값) |
| 1 ~ 7 | 중립톤 배지: "N일 전 기준" |
| 8 이상 | 경고톤 배지: "최근 거래 없음 · N일 전 호가" |

- **임계값: 7일** (8일째부터 경고톤). 주간 수집 주기 가정.
- 기존 "마지막 업데이트: {lastUpdatedAt}" 줄은 그대로 유지.
- 배지 톤 클래스는 기존 `changeChipClass` / 웜톤 테마 토큰 패턴을 따른다
  (raw hex 금지).

### 3. 차트 이월 구간 시각화 — `app/cards/[cardId]/_components/PriceHistoryChart.tsx`

마지막 실측점에서 오늘까지 추세선을 점선으로 연장:

- 실측 구간(마지막 관측일까지): 기존 실선.
- 이월 구간(마지막 관측일 → 오늘): 점선. 신규 데이터 없음을 명시.
- `stalenessDays === 0`이면 연장 구간 없음 (기존과 동일).
- 구현 세부(점선을 별도 series로 그릴지, 마지막 세그먼트만 dash 처리할지)는
  차트 라이브러리 API에 맞춰 구현 단계에서 확정. 요구사항은 "이월 구간이
  실측과 시각적으로 구분됨" 하나.

## 범위 밖 (YAGNI)

- DB 이월 행 (B안).
- 알림 신선도 게이팅 (오래된 값으로 트리거 방지). `stalenessDays` 필드가
  나중에 이를 가능케 하지만 이번엔 알림 동작을 바꾸지 않는다.
- `card_printings` 비정규화 최신가 컬럼 (C안 / Phase 3).

## 테스트

- `derivePriceDisplayFromHistory`: 주입한 `today`에 대해 `stalenessDays`가
  올바른지 (같은 날 = 0, 8일 전 실측 = 8). 기존 테스트에 케이스 추가.
- 배지 임계: 7일 = 중립, 8일 = 경고 경계 검증.

## 영향 파일

- `lib/tcg-catalog.ts` — `PriceDisplay` 필드, `derivePriceDisplayFromHistory` 시그니처+계산.
- `app/cards/[cardId]/page.tsx` — 신선도 배지.
- `app/cards/[cardId]/_components/PriceHistoryChart.tsx` — 점선 이월 구간.
- 관련 테스트 파일.
